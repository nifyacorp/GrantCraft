# Collaboration Features User Guide

This guide provides information on how to use GrantCraft's collaboration features to work with team members on grant proposals.

## Overview

The Collaboration features in GrantCraft enable teams to work together effectively on grant proposals by:

- Managing team member access and permissions
- Adding comments and feedback on specific files
- Tracking real-time presence of collaborators
- Sharing projects with external collaborators
- Monitoring activity logs to track changes

## Accessing Collaboration Features

1. Navigate to the Grants page
2. Click on the "Collaborate" tab in the main navigation
3. Select the project you want to collaborate on
4. Use the tabs to switch between Collaborators, Comments, and Activity views

## Managing Collaborators

The Collaborators tab allows you to invite team members and manage their permissions.

### Permission Levels

GrantCraft offers three permission levels:

- **Owner**: Can edit, share, and manage permissions
- **Editor**: Can edit and comment on documents
- **Viewer**: Can view and comment on documents, but not edit

### Adding Collaborators

1. In the Collaborators tab, locate the "Add Collaborator" section
2. Select a user from the dropdown menu
3. Choose the appropriate permission level
4. Click "Add Collaborator"

### Updating Permissions

1. Find the collaborator in the collaborators list
2. Use the permission dropdown menu to change their access level
3. Changes take effect immediately

### Removing Collaborators

1. Find the collaborator in the collaborators list
2. Click the "Remove" button
3. The user will immediately lose access to the project

### Sharing Projects

1. In the "Share Project" section, click "Generate Shareable Link"
2. The link will be copied to your clipboard
3. Share this link with collaborators who need to access the project
4. Links expire after 7 days by default

## Working with Comments

The Comments tab allows team members to discuss specific files and provide feedback.

### Adding Comments

1. Select a file from the "Active Files" panel
2. In the Comments tab, type your comment in the text box
3. Click "Add Comment"

### Replying to Comments

1. Find the comment you want to reply to
2. Click the "Reply" button
3. Enter your response
4. Click "Reply"

### Resolving Comments

1. When a comment has been addressed, click the "Resolve" button
2. Resolved comments can be hidden or shown using the "Show resolved" checkbox
3. Comments can be reopened if further discussion is needed

### Editing and Deleting Comments

1. You can edit or delete your own comments
2. Find the comment and click "Edit" or "Delete"
3. Project owners can delete any comments

## Real-time Collaboration

GrantCraft provides indicators to show who is currently working on a file.

1. When multiple users are viewing the same file, you'll see their avatars in the file header
2. This helps prevent conflicts and encourages coordination

## Activity Log

The Activity tab provides a chronological record of all project actions.

### Viewing Activities

1. Navigate to the Activity tab
2. Activities are shown in reverse chronological order (newest first)
3. Each activity shows the user, action, and timestamp

### Filtering Activities

1. Use the filter tabs to focus on specific activity types:
   - All Activities
   - Files (creations, updates, deletions)
   - Comments
   - Collaborators (permission changes, sharing)

## Best Practices for Collaboration

1. **Clear Roles**: Assign appropriate permissions based on team members' responsibilities
2. **Regular Updates**: Check the activity log to stay informed about project changes
3. **Detailed Comments**: Provide specific, constructive feedback in comments
4. **Resolved Comments**: Mark comments as resolved once the issues are addressed
5. **Coordinate Editing**: Use the presence indicators to avoid simultaneous editing
6. **Limited Sharing**: Only generate sharing links when needed, as they provide access to anyone with the link

## Troubleshooting

**Issue**: Can't add a collaborator
- Solution: Verify that you have owner permissions and that the user has an account in the system

**Issue**: Comments not appearing 
- Solution: Refresh the page and ensure you have selected a file

**Issue**: Collaborator can't access the project
- Solution: Check their permission level and regenerate the sharing link if needed

For additional help, contact system support.
