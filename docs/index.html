GrantCraft Documentation

Note: GrantCraft is an AI-powered grant writing assistant designed to help researchers develop compelling grant proposals through an intuitive interface.

GrantCraft provides researchers with a comprehensive set of tools to streamline the grant proposal creation process. Using a conversational AI interface, the platform helps users explore funding opportunities, structure their proposals, and generate high-quality content for each section of a grant application.

Key features include:

  • Conversational AI interface for grant exploration and planning
  • Intelligent document generation for all major grant sections
  • Budget and timeline creation tools
  • Project management and file organization
  • Version history tracking
  • User authentication and account management

Installation

GrantCraft is a web application built with React. Follow these steps to set up the development environment and install the necessary dependencies.

Prerequisites

  • Node.js (v14.0.0 or newer)
  • npm (v6.0.0 or newer)
  • Git

Setup Instructions

  1. Clone the repository:
Terminal 
git clone https://github.com/yourusername/grantcraft.git
cd grantcraft
  1. Install dependencies:
Terminal 
npm install
  1. Start the development server:
Terminal 
npm start

This will start the development server at http://localhost:1234 (or another port if 1234 is in use).

Quick Start

Here's a quick guide to get you started with GrantCraft:

1. Create an Account

Navigate to the application and create a new account by clicking "Sign In" and then "Register" on the authentication page. Fill in your details to create an account.

2. Start a New Project

After logging in, you'll be directed to the Grants page. Click "New Project" to start a new grant proposal. You can either use the chat interface to define your project or create it directly.

3. Use the Chat Interface

The chat interface helps you define your research project through a conversational experience. It will guide you through questions about your research topic, goals, and approach.

4. Generate Document Sections

Once your project is created, use the document generators to create different sections of your grant proposal:

  1. Navigate to your project's files page
  2. Click "Generate Document" and select the section you want to create (Abstract, Introduction, Methodology, etc.)
  3. Edit the generated content as needed
  4. Save your changes

5. Manage Your Files

Use the file manager to organize and edit your project files. You can view file history, restore previous versions, and organize content in appropriate folders.

User Accounts

GrantCraft includes a user authentication system that allows researchers to create accounts, save their work, and manage their grant proposals securely.

Registration

To register a new account:

  1. Navigate to the login page and click "Register"
  2. Fill in your details (username, email, password, and optional profile information)
  3. Submit the form to create your account

Login

To log in to an existing account:

  1. Navigate to the login page
  2. Enter your username/email and password
  3. Click "Login" to access your account

User Profile

Your user profile includes information that may be used in generated documents, such as your name and institution. This information helps personalize the grant documents the system creates for you.

Projects

Projects are the primary organizational unit in GrantCraft. Each project represents a grant proposal with its own files, sections, and settings.

Creating a Project

There are two ways to create a new project:

  1. Via Dashboard: Click "New Project" on the dashboard, then enter a title and research topic.
  2. Via Chat Interface: Start a new chat session and follow the prompts to define your research project through conversation.

Project Structure

Each project is organized into folders that correspond to different sections of a grant proposal:

  • abstract: Project abstract and executive summary
  • introduction: Project introduction and background
  • methodology: Research methodology and approach
  • budget: Budget and cost justification
  • timeline: Project timeline and milestones
  • impact: Project impact and significance
  • team: Research team information
  • supplementary: Supplementary materials and references

This structure helps organize your grant proposal in a logical way and ensures all essential sections are included.

Chat Interface

The chat interface is a key feature of GrantCraft that provides a conversational experience for developing your grant proposal. It guides you through the process of defining your research, identifying funding opportunities, and creating content.

Conversation Stages

The chat interface follows a structured conversation flow with several stages:

  1. Introduction: Initial greeting and explanation of how the assistant can help
  2. Topic Exploration: Discussion of research topic, goals, and approach
  3. Grant Matching: Identifying potential funding opportunities that match your research
  4. Project Creation: Setting up the project structure and basic information
  5. Section Development: Guidance on developing specific sections of the proposal

Using the Chat Interface

To use the chat interface effectively:

  • Provide detailed responses to questions about your research
  • Ask clarifying questions if you're unsure about what information is needed
  • Use commands like "create project" or "find grants" to trigger specific actions
  • Review suggestions and modify them as needed before accepting

The system will retain context from your conversation, allowing it to generate more relevant and personalized content for your grant documents.

Using Document Generators

To use a document generator:

  1. Navigate to your project's files page
  2. Click "Generate Document" in the toolbar
  3. Select the type of document you want to generate
  4. Review and edit the generated content
  5. Click "Save" to save the document to your project

Each generator provides section-specific controls and guidance to help you refine the content. You can regenerate individual sections or the entire document as needed.

File Management

GrantCraft provides a comprehensive file management system to organize your grant proposal documents. You can create, edit, and organize files within a logical folder structure.

File Organization

Files are organized into folders based on the standard sections of a grant proposal. Each folder contains relevant files for that section, making it easy to find and manage your content.

Creating and Editing Files

To create a new file:

  1. Navigate to the appropriate folder
  2. Click "New File" or "Add File"
  3. Enter a name for the file
  4. Add your content in the editor
  5. Save your changes

To edit an existing file:

  1. Click on the file in the file browser
  2. Make your changes in the editor
  3. Click "Save Changes" to save your edits

File Formats

GrantCraft primarily works with Markdown (.md) files for document content. Markdown provides a simple way to format text while maintaining readability. The system also supports other file types for supplementary materials.

Version History

GrantCraft includes a version history system that tracks changes to your documents, allowing you to view previous versions and restore them if needed.

Viewing Version History

To view the version history of a file:

  1. Open the file in the file viewer
  2. Click "View History" in the file footer
  3. Browse the available versions in the sidebar

Comparing Versions

The version history interface allows you to:

  • See when each version was created
  • View the content of each version
  • Compare changes between versions

Restoring Previous Versions

To restore a previous version:

  1. Select the version you want to restore in the version history view
  2. Click "Restore This Version"
  3. Confirm that you want to restore this version

The current version will be saved in the history before the restoration occurs, ensuring no work is lost.

Architecture

GrantCraft is built with a modular front-end architecture using React. The application is designed to be extensible and maintainable, with clear separation of concerns between components.

Core Architecture

The application follows a component-based architecture with these key layers:

  • UI Components: Reusable interface elements (buttons, cards, etc.)
  • Feature Components: Specialized components for specific features (ChatInterface, FileManager, etc.)
  • Pages: Container components that organize features (HomePage, GrantsPage)
  • Library Modules: Core functionality and business logic (projectManager, contextManager, etc.)
  • State Management: Handled through React hooks (useState, useEffect)

Data Management

Data in GrantCraft is managed through several key modules:

  • ProjectManager: Handles creation and management of projects and files
  • ContextManager: Manages conversation context and information extraction
  • DocumentGenerator: Generates document content based on templates and context
  • VersionManager: Tracks and manages document version history
  • AuthService: Handles user authentication and authorization

In the current implementation, data is stored in localStorage for simplicity. In a production environment, this would be replaced with a backend API and database.

Key Components

GrantCraft is built with a component-based architecture. Here are the key components and their responsibilities:

UI Components

  • ChatInterface: Provides the conversational AI experience
  • FileManager: Manages project files and folders
  • FileViewer: Displays and edits file content
  • VersionHistory: Shows file version history
  • Document Generators: Specialized components for generating different sections

Library Modules

  • ProjectManager: Manages projects and files
  • ContextManager: Manages conversation context
  • DocumentGenerator: Generates document content
  • VersionManager: Manages document versions
  • AuthService: Handles authentication

Component Communication

Components in GrantCraft communicate through:

  • Props: Parent components pass data and callbacks to children
  • Callbacks: Child components trigger actions in parents via function props
  • Service Instances: Shared services like ProjectManager are passed to components

Extending GrantCraft

GrantCraft is designed to be extensible. Here are some ways you can extend the application:

Adding New Document Generators

To add a new document generator:

  1. Create a new template in documentTemplates.js
  2. Create a new generator component similar to existing generators
  3. Update the FileManager component to include the new generator

Adding New Grant Templates

To add a new grant proposal template:

  1. Add a new template definition in projectManager.js under PROJECT_TEMPLATES
  2. Update the UI to allow selecting the new template

Backend Integration

To integrate GrantCraft with a backend:

  1. Create API services for each module (projects, files, auth, etc.)
  2. Replace local storage operations with API calls
  3. Implement authentication with JWT or session-based auth
  4. Add error handling and loading states for asynchronous operations

AI Integration

To integrate with real AI services:

  1. Update nlpConfig.js with API connection details
  2. Modify document generation to use actual API calls
  3. Implement retry logic and error handling for API failures

Deployment

GrantCraft can be deployed as a static site or with a backend API. Here are instructions for common deployment methods:

Building for Production

To build the application for production:

Terminal 
npm run build

This will create a dist directory with the built application.

Deploying to Netlify

GrantCraft includes a netlify.toml configuration for easy deployment to Netlify:

  1. Push your code to a Git repository
  2. Connect the repository to Netlify
  3. Configure the build command as npm run build
  4. Set the publish directory to dist
  5. Deploy

Other Deployment Options

The application can also be deployed to:

  • Vercel: Similar process to Netlify
  • GitHub Pages: Build locally and push to gh-pages branch
  • AWS S3 + CloudFront: For static hosting with CDN
  • Traditional hosting: Upload the dist directory to any web server
Note: For deployments with backend integration, you'll need to set up environment variables for API endpoints and other configuration.
