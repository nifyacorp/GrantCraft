Core Modules Reference

This document provides detailed information about the core modules that power GrantCraft. These modules contain the business logic and functionality that drives the application.

Note: This documentation is for developers who want to understand, modify, or extend the GrantCraft codebase.

ProjectManager Module

The ProjectManager module handles the creation, organization, and management of grant projects and their files. It provides functionality for creating project structures, managing files, and tracking changes.

Core Module Data Management

Classes

Project

Represents a single grant project with its structure, files, and metadata.

constructor(title, topic, templateId = 'standard', projectManager = null)

Creates a new Project instance.

Parameters

Name Type Description
title string The title of the project
topic string The research topic of the project
templateId string ID of the template to use (standard, nih, nsf)
projectManager ProjectManager Reference to the parent ProjectManager instance
Example
const project = new Project(
  'Novel Cancer Treatment Research',
  'Immunotherapy for Pancreatic Cancer',
  'nih'
);
createFile(folderName, fileName, content = '')

Creates a new file in the specified folder.

Parameters

Name Type Description
folderName string Name of the folder to create the file in
fileName string Name of the file to create
content string Initial content for the file

Returns

The newly created file object

Example
const file = project.createFile(
  'abstract',
  'abstract.md',
  '# Research Abstract\n\nThis is a sample abstract...'
);
updateFile(fileId, newContent)

Updates the content of a file and saves the previous version to history.

Parameters

Name Type Description
fileId string ID of the file to update
newContent string New content for the file

Returns

The updated file object

ProjectManager

Manages multiple Project instances, providing methods for creating, retrieving, and organizing projects.

constructor()

Creates a new ProjectManager instance.

Example
const projectManager = new ProjectManager();
createProject(title, topic, templateId = 'standard')

Creates a new project.

Parameters

Name Type Description
title string The title of the project
topic string The research topic of the project
templateId string ID of the template to use (standard, nih, nsf)

Returns

The newly created Project instance

Example
const project = projectManager.createProject(
  'Novel Cancer Treatment Research',
  'Immunotherapy for Pancreatic Cancer',
  'nih'
);
getProject(projectId)

Retrieves a project by its ID.

Parameters

Name Type Description
projectId string ID of the project to retrieve

Returns

The Project instance or null if not found

DocumentGenerator Module

The DocumentGenerator module is responsible for generating content for different sections of a grant proposal. It uses templates and natural language processing to create well-structured document content.

Core Module Content Generation

Classes

DocumentGenerator

Generates document content based on templates and conversation context.

constructor()

Creates a new DocumentGenerator instance.

Example
const documentGenerator = new DocumentGenerator();
async generateSection(sectionType, contextData, projectInfo)

Generates content for a specific document section.

Parameters

Name Type Description
sectionType string Type of section to generate (abstract, introduction, methodology, etc.)
contextData object Conversation context data
projectInfo object Project information (title, topic, etc.)

Returns

An object containing the generated content and metadata

Example
const abstract = await documentGenerator.generateSection(
  'abstract',
  contextData,
  { title: 'Cancer Research Project', topic: 'Immunotherapy' }
);
async generateSubsection(subsection, relevantContext, placeholderValues, sectionType)

Generates content for a specific subsection of a document.

Parameters

Name Type Description
subsection object Subsection template object
relevantContext object Extracted relevant context
placeholderValues object Values for template placeholders
sectionType string Type of section (abstract, introduction, etc.)

Returns

Generated content string for the subsection

ContextManager Module

The ContextManager module handles the management and processing of conversation context. It keeps track of conversation history and extracts relevant information for document generation.

Core Module Context Processing

Classes

ConversationContext

Manages conversation history and context for the chat interface.

constructor()

Creates a new ConversationContext instance.

Example
const context = new ConversationContext();
addMessage(message)

Adds a new message to the conversation history.

Parameters

Name Type Description
message object Message object with user or assistant content
Example
context.addMessage({
  role: 'user',
  content: 'I want to research cancer immunotherapy.'
});
extractResearchInfo()

Extracts research-related information from the conversation history.

Returns

Object containing extracted research information

AuthService Module

The AuthService module handles user authentication, authorization, and session management. It provides functions for user registration, login, and permission checking.

Core Module Authentication
Note: The current implementation uses mock data and local storage. In a production environment, this would connect to a real authentication API.

Methods

login(username, password)

Authenticates a user with username/email and password.

Parameters

Name Type Description
username string Username or email of the user
password string User's password

Returns

Promise resolving to the authenticated user object

Example
try {
  const user = await authService.login('researcher@example.com', 'password123');
  console.log('Logged in as:', user.firstName);
} catch (error) {
  console.error('Login failed:', error.message);
}
register(userData)

Registers a new user.

Parameters

Name Type Description
userData object User registration data

Returns

Promise resolving to the newly created user object

Example
try {
  const user = await authService.register({
    username: 'newresearcher',
    email: 'new@example.com',
    password: 'securepassword',
    firstName: 'Jane',
    lastName: 'Smith',
    institution: 'Research University'
  });
  console.log('Registered and logged in as:', user.username);
} catch (error) {
  console.error('Registration failed:', error.message);
}
logout()

Logs out the current user.

Returns

Promise resolving when logout is complete

Example
await authService.logout();
console.log('User logged out');
isAuthenticated()

Checks if a user is currently authenticated.

Returns

Boolean indicating authentication status

Example
if (authService.isAuthenticated()) {
  console.log('User is logged in');
} else {
  console.log('No authenticated user');
}
hasPermission(permission)

Checks if the current user has a specific permission.

Parameters

Name Type Description
permission string Permission to check

Returns

Boolean indicating if the user has the permission

Example
if (authService.hasPermission('create_project')) {
  console.log('User can create projects');
} else {
  console.log('User does not have permission to create projects');
}
